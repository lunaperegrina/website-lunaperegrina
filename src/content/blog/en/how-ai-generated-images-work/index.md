In recent years, images generated by artificial intelligence have started appearing **everywhere on the internet**.

Memes, fake photos, hyper-realistic portraits, political edits: many images that look photographic today may have been created by an AI model.

In this text, I will explain **in a simple way** how this process works and why some of these images are relatively easy to identify.

> Note: I am not an AI researcher. This is just an explanation based on curiosity and hands-on experimentation.

## What is Stable Diffusion?

One of the best-known image-generation models is **Stable Diffusion**.

It is a **deep learning** model capable of generating images from text.

This type of system is called **text-to-image (TXT2IMG)**.

You provide a prompt like:

```text
a cyberpunk city at night with neon lights
```

and the model generates a corresponding image.

It is conceptually similar to language models (like ChatGPT), but instead of **text -> text**, we have **text -> image**.

## Other usage modes

Besides **TXT2IMG**, these models can also work in other ways.

The two most common modes are:

### Text-to-Image (TXT2IMG)

Input: text
Output: a completely new image.

### Image-to-Image (IMG2IMG)

Input: an image + a prompt
Output: a new image **based on the original**, but modified.

This second mode is heavily used to transform existing images.

## The role of noise

Diffusion models are based on one central concept: **noise**.

During training, the model learns to:

1. progressively add noise to images
2. then **remove that noise to reconstruct a plausible image**

During image generation, the process is reversed.

The model starts from something close to **random noise** and gradually transforms that noise into a coherent image guided by the prompt.

## How IMG2IMG works

In **image-to-image** mode, the original image is partially transformed into noise.

After that, the model reconstructs the image guided by the provided prompt.

The transformation level is controlled by a parameter called **denoise strength**.

## Denoise control

The **denoise** parameter controls how much of the original image is preserved.

- **low denoise** -> the final image stays very close to the original
- **high denoise** -> the image can change a lot

This allows things like:

- changing artistic style
- modifying expressions or poses
- turning photos into paintings
- generating variations of an existing image

## Why do some images look "obviously AI"?

When an image is generated from another one (for example, with IMG2IMG), the model often preserves:

- **silhouettes**
- **facial structure**
- **object positions**

This happens because the model is still using the **geometry of the original image as a base**.

In many cases, the original image is used only as **structure**, while the rest is reconstructed by AI.

## Other techniques

Beyond the base diffusion method, there are complementary techniques such as:

- **Textual Inversion**
- **DreamBooth**
- **LoRA**

These techniques allow the model to be trained to reproduce:

- specific styles
- characters
- faces
- visual identities

That makes generated images even more convincing.

## The verification problem

Today it is already **hard to distinguish real images from AI-generated images**, especially when:

- there is post-processing
- the model was well trained
- resolution is high

As these models improve, this challenge is likely to become even more complex.
