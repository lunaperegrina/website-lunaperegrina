# Suas senhas provavelmente são ruins (e eu posso provar)

A maioria das pessoas passou a vida inteira criando **senhas da forma errada**.

Aquelas regras clássicas tipo:

- colocar números
- adicionar caracteres especiais
- misturar maiúsculas e minúsculas
- limitar a quantidade de caracteres

Na prática, isso **não resolve o problema principal** de segurança de senhas.

O conceito que realmente importa é outro: **entropia de senha**.

## O que é entropia de senha?

**Entropia de senha** é uma medida da **aleatoriedade e complexidade de uma senha**.

Quanto maior a entropia:

- mais combinações possíveis existem
- mais difícil fica quebrar a senha
- maior o tempo necessário para um ataque de força bruta

Ou seja: entropia mede **o quão difícil seria adivinhar ou quebrar uma senha**.

## Senhas "complexas" nem sempre são melhores

Considere dois exemplos:

```

Tr0ub4dor&3

```
```

cavalo amarelo dança na chuva

```

A primeira parece mais "segura" porque tem:

- números
- símbolos
- mistura de caracteres

Mas na prática ela pode ter **menos entropia** do que a segunda.

Isso acontece porque **senhas curtas têm menos combinações possíveis**, mesmo com símbolos.

Já uma frase longa aumenta drasticamente o espaço de busca.

## Como verificar a entropia de uma senha

Existem calculadoras online que estimam a entropia de uma senha.

Um exemplo:

https://apps.cygnius.net/passtest/

Essas ferramentas mostram:

- entropia estimada
- tempo aproximado para quebrar a senha
- resistência contra diferentes tipos de ataque

Elas deixam claro que **comprimento costuma importar mais que complexidade visual**.

## O problema das senhas cheias de símbolos

Senhas do tipo:

```

T$9!xQ#4

```

têm um problema sério:

- são difíceis de lembrar
- são difíceis de digitar
- geram frustração

O que as pessoas fazem então?

- salvam em um bloco de notas
- reutilizam a mesma senha
- armazenam em lugares inseguros

Ou seja, **a segurança teórica vira vulnerabilidade prática**.

## A abordagem melhor: passphrases

Uma estratégia muito mais eficiente é usar **passphrases**.

Passphrases são **frases longas formadas por palavras aleatórias**.

Exemplo:

```

gato laranja dorme no teclado

```

Vantagens:

- alta entropia
- fácil de lembrar
- fácil de digitar
- menor chance de erro

Quanto mais palavras, maior a entropia.

## E quando o site exige números ou símbolos?

A solução é simples: **adicione sem quebrar a frase**.

Por exemplo:

```

gato laranja dorme no teclado 42!

```

ou

```

gato_laranja_dorme_no_teclado

```

O importante é **manter o comprimento da senha alto**.

## Como saber se sua senha é ruim

Provavelmente sua senha é fraca se ela:

- é curta
- segue padrões previsíveis
- foi criada para cumprir regras arbitrárias
- é difícil de lembrar

Se você precisa **anotar sua senha para não esquecer**, isso já é um sinal de problema.

## Regra prática

Uma boa senha geralmente tem:

- **comprimento alto**
- **palavras fáceis de lembrar**
- **estrutura imprevisível**

Algo como:

```

chuva roxa atravessa a montanha 2040

```

é muito mais seguro do que:

```

P@55w0rd!

```

Leve isso em consideração na próxima vez que criar ou atualizar suas senhas :D
